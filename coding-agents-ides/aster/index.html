Aster - Your AI CoPilot on Mobile

Your AI CoPilot for any Android device using MCP (Model Context Protocol) ‚Äî or give your AI a dedicated phone and let it call, text, and act on its own. Fully open source and privacy-first ‚Äî your data never leaves your network.

Website: aster.theappstack.in | GitHub: github.com/satyajiit/aster-mcp

For screenshots of the Android app and web dashboard, visit aster.theappstack.in.

Setup

  1. Install and start the server:
npm install -g aster-mcp
aster start

  1. Install the Aster Android app on any Android device ‚Äî your daily phone or a spare one for your AI ‚Äî from Releases and connect to the server address shown in terminal.

  2. Configure MCP in your .mcp.json:

{
  "mcpServers": {
    "aster": {
      "type": "http",
      "url": "http://localhost:5988/mcp"
    }
  }
}

Security & Privacy

Aster is built with a security-first, privacy-first architecture:

  • Self-Hosted ‚Äî Runs entirely on your local machine. No cloud servers, no third-party relays. Your data stays on your network.
  • Zero Telemetry ‚Äî No analytics, no tracking, no usage data collection. What you do stays with you.
  • Device Approval ‚Äî Every new device must be manually approved from the dashboard before it can connect or execute commands.
  • Tailscale Integration ‚Äî Optional encrypted mesh VPN via Tailscale with WireGuard. Enables secure remote access with automatic TLS (WSS) ‚Äî no port forwarding required.
  • No Root Required ‚Äî Uses the official Android Accessibility Service API (same system powering screen readers). No rooting, no ADB hacks, no exploits. Every action is permission-gated and sandboxed.
  • Foreground Transparency ‚Äî Always-visible notification on your Android device when the service is running. No silent background access.
  • Local Storage Only ‚Äî All data (device info, logs) stored in a local SQLite database. Nothing is sent externally.
  • 100% Open Source ‚Äî MIT licensed, fully auditable codebase. Inspect every line of code on GitHub.

Proactive Event Forwarding (OpenClaw Callbacks)

Aster can push real-time events from the phone to your AI agent via webhook. When enabled, these events arrive as HTTP POST payloads ‚Äî your agent doesn't need to poll, the phone tells you what's happening.

Configure via the dashboard at /settings/openclaw or CLI: aster set-openclaw-callbacks.

Webhook Format

Events are sent as HTTP POST to the configured OpenClaw endpoint (/hooks/agent by default). The AI reads the message field. All event context is packed into message using standardized [key] value tags.

Example raw HTTP POST payload for a notification event:

{
  "message": "[skill] aster\n[event] notification\n[device_id] 6241e40fb71c0cf7\n[model] samsung SM-S938B, Android 16\n[data-app] messaging\n[data-package] com.google.android.apps.messaging\n[data-title] John\n[data-text] Hey, are you free tonight?",
  "wakeMode": "now",
  "deliver": true,
  "channel": "whatsapp",
  "to": "+1234567890"
}
  • message ‚Äî structured event text with standard headers (this is what the AI reads)
  • wakeMode ‚Äî always "now" (wake the agent immediately)
  • deliver ‚Äî always true for real events, false for test pings
  • channel / to ‚Äî delivery channel and recipient, configured in the dashboard

Event Format

Every event follows a standardized structure with 4 fixed headers and [data-*] fields:

[skill] aster
[event] <event_name>
[device_id] <device_uuid>
[model] <manufacturer model, Android version>
[data-key] value
[data-key] value
  • [skill] ‚Äî always aster
  • [event] ‚Äî event name: sms, notification, device_online, device_offline, pairing
  • [device_id] ‚Äî UUID of the device (use this to target the device with Aster tools)
  • [model] ‚Äî device manufacturer, model, and OS
  • [data-*] ‚Äî event-specific fields, each prefixed with data- (e.g. [data-app], [data-sender])

Event Types

sms ‚Äî Incoming SMS

[skill] aster
[event] sms
[device_id] a1b2c3d4-5678-90ab
[model] samsung SM-S938B, Android 15
[data-sender] +1234567890
[data-body] Hey are you free tonight?

notification ‚Äî App notification (deduplicated against SMS)

[skill] aster
[event] notification
[device_id] a1b2c3d4-5678-90ab
[model] samsung SM-S938B, Android 15
[data-app] whatsapp
[data-package] com.whatsapp
[data-title] John
[data-text] Meeting moved to 3pm

device_online ‚Äî Approved device came online

[skill] aster
[event] device_online
[device_id] a1b2c3d4-5678-90ab
[model] samsung SM-S938B, Android 15
[data-status] connected

device_offline ‚Äî Device went offline

[skill] aster
[event] device_offline
[device_id] a1b2c3d4-5678-90ab
[model] samsung SM-S938B, Android 15
[data-status] disconnected

pairing ‚Äî New device needs approval (use [device_id] to approve)

[skill] aster
[event] pairing
[device_id] e5f6g7h8-9012-cdef
[model] Samsung SM-S924B, Android 15
[data-status] pending_approval
[data-action] approve this device from the Aster dashboard or via aster devices approve

How to React to Events

When you receive a message with [skill] aster, parse the [event] and [device_id] to determine what happened and which device to act on.

SMS ‚Äî reply, extract info, or escalate:

[event] sms | [device_id] a1b2c3d4 | sender: +1234567890 | body: Running late, be there in 20
‚Üí aster_send_sms (deviceId: a1b2c3d4) to +1234567890: "No worries, see you soon!"

[event] sms | [device_id] a1b2c3d4 | sender: +1800555 | body: Your OTP is 482913
‚Üí Extract OTP "482913", use aster_input_text (deviceId: a1b2c3d4) to enter it

Notifications ‚Äî monitor and act on behalf of user:

[event] notification | [device_id] a1b2c3d4 | app: driver | text: Your driver is arriving
‚Üí aster_speak_tts (deviceId: a1b2c3d4) "Your Uber is almost here"

[event] notification | [device_id] a1b2c3d4 | app: mShop | text: Your package was delivered
‚Üí aster_send_sms (deviceId: a1b2c3d4) to user: "Your Amazon package just arrived"

Device lifecycle ‚Äî manage connectivity:

[event] device_offline | [device_id] a1b2c3d4
‚Üí Pause pending automations for device a1b2c3d4

[event] device_online | [device_id] a1b2c3d4
‚Üí Resume queued tasks, aster_read_notifications (deviceId: a1b2c3d4) to catch up

Pairing ‚Äî approve or alert:

[event] pairing | [device_id] e5f6g7h8 | model: Samsung SM-S924B
‚Üí If expected: approve device e5f6g7h8 via dashboard API
‚Üí If unexpected: alert user "Unknown device SM-S924B trying to connect"

Commands

aster start              # Start the server
aster stop               # Stop the server
aster status             # Show server and device status
aster dashboard          # Open web dashboard

aster devices list       # List connected devices
aster devices approve    # Approve a pending device
aster devices reject     # Reject a device
aster devices remove     # Remove a device

aster set-openclaw-callbacks  # Configure event forwarding to OpenClaw

Requirements

  • Node.js >= 20
  • Any Android device with Aster app installed (your phone or a dedicated AI device)
  • Device and server on same network (or use Tailscale for secure remote access)
