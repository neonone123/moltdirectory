Ad-Ready: AI Advertising Image Generator

Generate professional advertising images from product URLs using a 4-phase AI pipeline on ComfyDeploy.

‚ö†Ô∏è CRITICAL: Required Inputs Checklist

Before running ANY ad generation, the agent MUST ensure ALL of these are provided:

Input Required? How to Get It
--product-url ‚úÖ ALWAYS User provides the product page URL
--product-image ‚úÖ ALWAYS Download from the product page, or user provides
--logo ‚úÖ ALWAYS Download from brand website or search online. MUST be an image file
--reference ‚úÖ RECOMMENDED An existing ad whose style we want to clone. Search online or use previously generated images
--brand-profile ‚úÖ NEVER EMPTY Pick from catalog or run brand-analyzer first. NEVER leave as "No Brand" if a brand is known
--prompt-profile ‚úÖ ALWAYS Choose based on campaign objective
--aspect-ratio Default: 4:5 Change if needed for platform
--model Optional Model/talent face from catalog or user-provided

üö® NEVER Skip These Steps:

  1. Product image ‚Äî Download the main product photo from the product URL. The scraper is fragile; always provide a product image explicitly.
  2. Brand logo ‚Äî Download the logo from the brand's official website or search for "{brand name} logo" online. Must be a clean logo image (PNG preferred).
  3. Brand profile ‚Äî If the brand doesn't exist in the catalog, run brand-analyzer skill FIRST to generate one. Never submit with "No Brand" when a brand is known.
  4. Reference image ‚Äî Search for an existing ad or visual with a style that matches what we're generating. Can be from previously generated images, the brand's campaigns, or found online. This dramatically improves output quality.

Auto-Preparation Workflow

When the user asks to generate an ad, follow this workflow:

1. User provides: product URL + brand name + objective

2. CHECK brand profile exists:
   ‚Üí ls ~/clawd/ad-ready/configs/Brands/ | grep -i "{brand}"
   ‚Üí If not found: run brand-analyzer skill first
   
3. DOWNLOAD product image:
   ‚Üí Visit the product URL in browser or fetch the page
   ‚Üí Find and download the main product image
   ‚Üí Save to /tmp/ad-ready-product.jpg

4. DOWNLOAD brand logo:
   ‚Üí Search "{brand name} logo PNG" or fetch from brand website
   ‚Üí Download clean logo image
   ‚Üí Save to /tmp/ad-ready-logo.png

5. FIND reference image:
   ‚Üí Search for "{brand name} advertisement" or similar
   ‚Üí Or use a previously generated ad that has the right style
   ‚Üí Save to /tmp/ad-ready-reference.jpg

6. SELECT prompt profile based on objective:
   ‚Üí Awareness: brand discovery, first impressions
   ‚Üí Interest: engagement, curiosity
   ‚Üí Consideration: comparison, features
   ‚Üí Evaluation: deep dive, decision support
   ‚Üí Conversion: purchase intent, CTAs (most common)
   ‚Üí Retention: re-engagement
   ‚Üí Loyalty: brand advocates
   ‚Üí Advocacy: referral, community

7. RUN the generation with ALL inputs filled

Usage

Full command (recommended):

COMFY_DEPLOY_API_KEY="$KEY" uv run ~/.clawdbot/skills/ad-ready/scripts/generate.py \
  --product-url "https://shop.example.com/product" \
  --product-image "/tmp/product-photo.jpg" \
  --logo "/tmp/brand-logo.png" \
  --reference "/tmp/reference-ad.jpg" \
  --model "models-catalog/catalog/images/model_15.jpg" \
  --brand-profile "Nike" \
  --prompt-profile "Master_prompt_05_Conversion" \
  --aspect-ratio "4:5" \
  --output "ad-output.png"

Auto-fetch mode (downloads product image and logo automatically):

COMFY_DEPLOY_API_KEY="$KEY" uv run ~/.clawdbot/skills/ad-ready/scripts/generate.py \
  --product-url "https://shop.example.com/product" \
  --brand-profile "Nike" \
  --prompt-profile "Master_prompt_05_Conversion" \
  --auto-fetch \
  --output "ad-output.png"

The --auto-fetch flag will:

  • Download the main product image from the product URL
  • Search and download the brand logo
  • Both get uploaded to ComfyDeploy automatically

4-Phase Pipeline (How It Works Internally)

Phase 1: Product Scraping

  • Gemini Flash visits the product URL
  • Extracts: title, description, features, price, images
  • ‚ö†Ô∏è Image scraping is the most fragile part ‚Äî always provide product images manually

Phase 2: Campaign Brief Generation (CRITICAL)

  • Uses Brand Identity JSON + Product Data ‚Üí 10-point brief
  • Everything downstream depends on brief quality
  • Brief covers: strategic objective, central message, visual tone, product role, photographer, art direction, environment, textures, signature

Phase 3: Blueprint Generation

  • Master Prompt (per funnel stage) + Brief + Product JSON + Keyword Bank + Format
  • Gemini Flash generates complete Blueprint JSON
  • Covers: scene, production, graphic design, lighting, composition, materials, CTA

Phase 4: Image Generation

  • Nano Banana Pro (Imagen 3.0) generates the final image
  • Uses Blueprint JSON + all reference images (product, talent, logo, style ref)

Supporting Reference Nodes

  • pose_ref ‚Üí enforce a specific pose (replicated exactly)
  • photo_style_ref ‚Üí replicate photographic style (‚ö†Ô∏è can be too literal, being optimized)
  • location_ref ‚Üí replicate location and color palette

Brand Profiles

ls ~/clawd/ad-ready/configs/Brands/*.json | sed 's/.*\///' | sed 's/\.json//'

Creating new brand profiles:

Use the brand-analyzer skill:

GEMINI_API_KEY="$KEY" uv run ~/.clawdbot/skills/brand-analyzer/scripts/analyze.py \
  --brand "Brand Name" --auto-save

This generates a full Brand Identity JSON and saves it to the catalog automatically.

Known Limitations

  1. Product image scraping is fragile ‚Äî always provide product images manually when possible
  2. photo_style_ref can be too literal ‚Äî the style reference may be replicated too closely
  3. Some websites block scraping ‚Äî Armani works well, others may return incorrect data
  4. Auto 4-Format is alpha ‚Äî bugs and edge cases exist
  5. Gemini hallucinations ‚Äî occasional issues in complex reasoning steps
