whoopskill

Use whoopskill to fetch WHOOP health metrics (sleep, recovery, HRV, strain, workouts).

Install: npm install -g whoopskill | GitHub

Quick start

  • whoopskill summary ‚Äî one-liner: Recovery: 52% | HRV: 39ms | Sleep: 40% | Strain: 6.7
  • whoopskill summary --color ‚Äî color-coded summary with üü¢üü°üî¥ status indicators
  • whoopskill trends ‚Äî 7-day trends with averages and direction arrows
  • whoopskill trends --days 30 --pretty ‚Äî 30-day trend analysis
  • whoopskill insights --pretty ‚Äî AI-style health recommendations
  • whoopskill --pretty ‚Äî human-readable output with emojis
  • whoopskill recovery ‚Äî recovery score, HRV, RHR
  • whoopskill sleep ‚Äî sleep performance, stages
  • whoopskill workout ‚Äî workouts with strain
  • whoopskill --date 2025-01-03 ‚Äî specific date

Analysis commands

  • summary ‚Äî quick health snapshot (add --color for status indicators)
  • trends ‚Äî multi-day averages with trend arrows (‚Üë‚Üì‚Üí)
  • insights ‚Äî personalized recommendations based on your data

Data types

  • profile ‚Äî user info (name, email)
  • body ‚Äî height, weight, max HR
  • sleep ‚Äî sleep stages, efficiency, respiratory rate
  • recovery ‚Äî recovery %, HRV, RHR, SpO2, skin temp
  • workout ‚Äî strain, HR zones, calories
  • cycle ‚Äî daily strain, calories

Combine types

  • whoopskill --sleep --recovery --body

Auth

  • whoopskill auth login ‚Äî OAuth flow (opens browser)
  • whoopskill auth status ‚Äî check token status
  • whoopskill auth logout ‚Äî clear tokens

Notes

  • Output is JSON to stdout (use --pretty for human-readable)
  • Tokens stored in ~/.whoop-cli/tokens.json (auto-refresh)
  • Uses WHOOP API v2
  • Date follows WHOOP day boundary (4am cutoff)
  • WHOOP apps with <10 users don't need review (immediate use)
